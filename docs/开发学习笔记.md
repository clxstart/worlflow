# WorkFlow 项目开发学习笔记

本文档记录从零开始开发一个 Spring Boot 项目的完整流程，按照实际开发顺序逐步讲解。

## 一、项目初始化

### 1.1 创建项目

使用 Spring Initializr 创建项目，选择：
- Spring Boot 2.7.18（兼容 Java 8）
- Java 8
- Maven 构建

### 1.2 配置 pom.xml 依赖

```xml
<properties>
    <java.version>1.8</java.version>
    <spring-boot.version>2.7.18</spring-boot.version>
    <mybatis-plus.version>3.5.3.1</mybatis-plus.version>
    <hutool.version>5.8.11</hutool.version>
    <fastjson.version>2.0.16</fastjson.version>
    <jwt.version>0.11.5</jwt.version>
</properties>
```

**核心依赖：**
| 依赖 | 作用 |
|------|------|
| spring-boot-starter-web | Web 开发 |
| spring-boot-starter-security | 安全认证 |
| mybatis-plus-boot-starter | ORM 框架 |
| spring-boot-starter-data-redis | Redis 缓存 |
| mysql-connector-java | MySQL 驱动 |
| jjwt-api | JWT Token |

### 1.3 配置 application.yml

```yaml
# 主配置
spring:
  profiles:
    active: local

# MyBatis-Plus 配置
mybatis-plus:
  type-aliases-package: com.clx.workflow.**.domain
  mapper-locations: classpath*:mapper/**/*Mapper.xml

# Token 配置
token:
  header: Authorization
  secret: your-secret-key
  expireTime: 30
```

## 二、基础框架搭建

### 2.1 开发顺序

```
1. HttpStatus（状态码常量）
      ↓
2. StringUtils（字符串工具类）
      ↓
3. AjaxResult（统一响应格式）
      ↓
4. BaseController（基础控制器）
      ↓
5. GlobalExceptionHandler（全局异常处理）
```

### 2.2 统一响应格式 AjaxResult

所有 API 返回统一的 JSON 格式：

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": { ... }
}
```

**核心代码：**

```java
public class AjaxResult extends HashMap<String, Object> {
    public static final String CODE_TAG = "code";
    public static final String MSG_TAG = "msg";
    public static final String DATA_TAG = "data";

    public static AjaxResult success() {
        return new AjaxResult(HttpStatus.SUCCESS, "操作成功");
    }

    public static AjaxResult error(String msg) {
        return new AjaxResult(HttpStatus.ERROR, msg);
    }
}
```

### 2.3 全局异常处理器

使用 `@RestControllerAdvice` 捕获所有异常：

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(ServiceException.class)
    public AjaxResult handleServiceException(ServiceException e) {
        return AjaxResult.error(e.getMessage());
    }

    @ExceptionHandler(Exception.class)
    public AjaxResult handleException(Exception e) {
        return AjaxResult.error(e.getMessage());
    }
}
```

```java
public class SysUser extends BaseEntity {

    @TableId(type = IdType.AUTO)
    private Long userId;

    private String userName;
    private String nickName;
    private String password;
    private String status;

    // getter/setter...

```

### 4.3 Mapper 接口

```java
@Mapper
public interface SysUserMapper extends BaseMapper<SysUser> {
    // BaseMapper 已提供 CRUD 方法
}
```

### 4.4 Service 层

```java
public interface ISysUserService extends IService<SysUser> {
    SysUser selectUserByUserName(String userName);
}

@Service
public class SysUserServiceImpl extends ServiceImpl<SysUserMapper, SysUser>
    implements ISysUserService {

    @Override
    public SysUser selectUserByUserName(String userName) {
        return this.lambdaQuery()
                .eq(SysUser::getUserName, userName)
                .one();
    }
}
```

### 4.5 Controller 层

```java
@RestController
@RequestMapping("/system/user")
public class SysUserController extends BaseController {

    @Autowired
    private ISysUserService userService;

    @GetMapping("/list")
    public AjaxResult list(SysUser user,
                           @RequestParam(defaultValue = "1") Integer pageNum,
                           @RequestParam(defaultValue = "10") Integer pageSize) {
        Page<SysUser> page = userService.lambdaQuery()
                .like(user.getUserName() != null, SysUser::getUserName, user.getUserName())
                .page(new Page<>(pageNum, pageSize));
        return success(page);
    }
}
```

## 五、JWT 登录认证

### 5.1 认证流程图

```
┌──────────────┐     POST /login      ┌──────────────┐
│   客户端      │ ───────────────────→ │  Controller  │
│              │    {username, pwd}   │              │
└──────────────┘                      └──────────────┘
                                             │
                                             ↓
                                      ┌──────────────┐
                                      │   Service    │
                                      │              │
                                      └──────────────┘
                                             │
              ┌──────────────────────────────┼────────────────────────┐
              ↓                              ↓                        ↓
       ┌──────────────┐              ┌──────────────┐        ┌──────────────┐
       │   MySQL      │              │    Redis     │        │   JWT 生成   │
       │  (用户数据)   │              │  (Token缓存) │        │              │
       └──────────────┘              └──────────────┘        └──────────────┘
              ↑                              ↑                        │
              └──────────────────────────────┴────────────────────────┘
                                             │
                                             ↓
                                      返回 Token 给客户端
```

### 5.2 登录流程详解

**步骤1：接收请求**
```
SysLoginController.login()
    ↓
SysLoginService.login(username, password)
```

**步骤2：验证用户**
```
authenticationManager.authenticate(
    new UsernamePasswordAuthenticationToken(username, password)
)
    ↓
UserDetailsServiceImpl.loadUserByUsername("admin")
    ↓
从数据库查询用户 → userService.selectUserByUserName("admin")
    ↓
返回 LoginUser 对象
```

**步骤3：生成 Token**
```
TokenService.createToken(loginUser)
    ↓
1. 生成 UUID 作为 token 标识
2. 创建 JWT（包含 UUID）
3. 存入 Redis：key = "login_tokens:{uuid}", value = LoginUser
    ↓
返回 Token
```

### 5.3 访问受保护接口

**请求：**
```
GET /system/user/list
Header: Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

**执行流程：**
```
JwtAuthenticationTokenFilter.doFilterInternal()
    ↓
TokenService.getLoginUser(request)
    ↓
1. 从 Header 获取 Token
2. 解析 JWT，获取 UUID
3. 从 Redis 获取 LoginUser
    ↓
TokenService.verifyToken(loginUser)  // 检查过期时间
    ↓
存入 SecurityContextHolder
    ↓
请求继续 → Controller
```

### 5.4 核心代码

**TokenService：**

```java
@Component
public class TokenService {

    // 生成 Token
    public String createToken(LoginUser loginUser) {
        String token = UUID.randomUUID().toString();
        loginUser.setToken(token);
        refreshToken(loginUser);  // 存入 Redis

        Map<String, Object> claims = new HashMap<>();
        claims.put("login_user_key", token);
        return Jwts.builder()
                .setClaims(claims)
                .signWith(getSignKey(), SignatureAlgorithm.HS256)
                .compact();
    }

    // 获取登录用户
    public LoginUser getLoginUser(HttpServletRequest request) {
        String token = getToken(request);
        Claims claims = parseToken(token);
        String uuid = (String) claims.get("login_user_key");
        return redisCache.getCacheObject("login_tokens:" + uuid);
    }
}
```

**Security 配置：**

```java
@Configuration
@EnableGlobalMethodSecurity(prePostEnabled = true)
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .csrf().disable()
            .sessionManagement().sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            .and()
            .authorizeRequests()
            .antMatchers("/login", "/register").anonymous()
            .anyRequest().authenticated();

        // 添加 JWT 过滤器
        http.addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);
    }
}
```

## 十、角色管理模块

### 10.1 数据库设计

**角色表 sys_role：**

```sql
CREATE TABLE `sys_role` (
    `role_id`       BIGINT NOT NULL AUTO_INCREMENT COMMENT '角色ID',
    `role_name`     VARCHAR(30) NOT NULL COMMENT '角色名称',
    `role_key`      VARCHAR(100) NOT NULL COMMENT '角色权限字符串',
    `role_sort`     INT NOT NULL COMMENT '显示顺序',
    `status`        CHAR(1) NOT NULL DEFAULT '0' COMMENT '角色状态（0正常 1停用）',
    `deleted`       INT DEFAULT 0 COMMENT '删除标志',
    `create_time`   DATETIME COMMENT '创建时间',
    `update_time`   DATETIME COMMENT '更新时间',
    `remark`        VARCHAR(500) COMMENT '备注',
    PRIMARY KEY (`role_id`)
);
```

**用户角色关联表 sys_user_role：**

```sql
CREATE TABLE `sys_user_role` (
    `user_id`   BIGINT NOT NULL COMMENT '用户ID',
    `role_id`   BIGINT NOT NULL COMMENT '角色ID',
    PRIMARY KEY (`user_id`, `role_id`)
);
```

### 10.2 分层结构

```

      ↓

SysRoleController.java # 角色控制器
```

### 10.3 核心功能

| 功能 | 接口 | 方法 |
|------|------|------|
| 角色列表 | `/system/role/list` | GET |
| 所有角色 | `/system/role/all` | GET |
| 角色详情 | `/system/role/{id}` | GET |
| 新增角色 | `/system/role` | POST |
| 修改角色 | `/system/role` | PUT |
| 删除角色 | `/system/role/{ids}` | DELETE |
| 授权用户 | `/system/role/authUser/selectAll` | PUT |

### 10.4 角色与权限

角色权限通过 `role_key` 字段标识，用于 Spring Security 权限控制：

```java
@PreAuthorize("@ss.hasPermi('system:role:list')")
@GetMapping("/list")
public AjaxResult list(...) { ... }
```

**权限字符串格式**：`模块:功能:操作`

示例：
- `system:role:list` - 角色列表权限
- `system:role:add` - 新增角色权限
- `system:role:edit` - 修改角色权限
- `system:role:remove` - 删除角色权限

## 十一、学习要点总结

1. **分层架构**：Controller → Service → Mapper，职责分离
2. **统一响应**：AjaxResult 统一返回格式
3. **全局异常**：@RestControllerAdvice 统一处理异常
4. **JWT 认证**：无状态认证，Token 存 Redis
5. **Spring Security**：过滤器链实现权限控制
6. **MyBatis-Plus**：简化 CRUD 操作
7. **角色权限**：role_key 配合 @PreAuthorize 实现权限控制
